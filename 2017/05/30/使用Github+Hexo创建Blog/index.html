使用Github+Hexo创建Blog

发表于 2017-05-30

date: 2017-03-12 16:54:37

前言

越来越多的程序员使用Github来寻找或搭建自己感兴趣的项目，协作式的软件开发方法成为了软件开发人员应掌握的基本技能。其实Github的功能还有很多，比如使用Github来搭建自己的Blog，记录自己的学习过程，与他人分享自己的经验与见解。下面将展现使用Github+Hexo搭建个人博客的过程。

准备工具

Github

GitHub，是一个代码托管网站和社交编程网站。我们需要注册一个账号。

Github Pages

GitHub Pages，是用来托管 GitHub 上静态网页的免费站点，那 GitHub Pages具体有哪些功能呢：

  • 有足够的免费空间（1G），资料自己管理，保存可靠；
  • 享受 GitHub 的便利，上面有很多大牛，眼界会开阔很多；
  • 可以使用很多现成的博客框架，搭建过程简单快速;

Hexo

Hexo是快速、简洁且高效的博客框架。它具有以下优势：

  • 超快速度：Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。
  • 支持 Markdown：Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。
  • 一键部署：只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。
  • 丰富的插件：Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。

使用 GitHub Pages 建立博客站点

注册 GitHub

访问 GitHub，注册十分简单，一定要记住注册时使用的邮箱，因为 GitHub 上很多通知都是通过邮箱的。如下图所示：

在 GitHub 上建立仓库

与 GitHub 建立好连接之后，就可以方便的使用它提供的 Pages 服务。创建新的仓库。

安装软件

安装Node.js、GitHub for Windows

配置 SSH

让本地 git 项目与远程的 GitHub 建立联系，使用ssh。如果机器中没有ssh工具需要自己安装，输入如下命令：

ssh -T git@github.com

命令完成后会有成功的反馈。

使用 Hexo 创建博客框架

Hexo 安装，打开命令行，依次输入以下命令：

npm install hexo-cli -g

Hexo部署

切换到想要建立的目录，输入如下命令：hexo init Hexo，部署成功之后，Hexo 会自动在目标文件夹建立博客网站所需要的所有文件。可使用如下命令进行本地预览hexo generate hexo server。此时打开浏览器，在浏览器地址栏输入 http://localhost:4000/ （默认端口为4000）, 便可以看到最原始的博客了。

将本地文件部署到 GitHub

发表博文

我们可以使用命令新建一篇博文,使用 Git Shell 进入 Hexo 文件夹，输入以下命令：

hexo new "文章题目"

命令执行完后，就会发现在

Hexo\source_posts

目录中多了一个文件博文名.md，这就是我们刚才新建的博文。

此外，我们也可以直接进入

Hexo\source_posts

目录中，右键新建一个文本文档，将名字改为博文名.md，这样也新建了一篇博文。

博文写好了，你得发表出去别人才看得到呀。依然在 Git Shell 中进入 Hexo 文件夹，执行下面几条命令，将博客部署到 GitHub 上：

hexo clean hexo generate hexo deploy

每次更新博文或者发布新的博文都需要执行以上的命令。
